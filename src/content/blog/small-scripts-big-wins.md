Not every useful program is a project. Some of the best code I've written
this year is under ten lines long.

A script that renames a folder of exported files by date. A one-liner that
checks whether two CSVs have the same columns before I merge them. A tiny
alias that opens today's notes file.

None of these are impressive. All of them get used every single week, which
is more than I can say for most of my ambitious projects.

The lesson I keep relearning: optimize the things you actually do, not the
things you imagine doing.
